1. - # HTTP 请求头详解

     浏览器发送 HTTP 请求时，会在请求头（Request Headers）中附加关键元数据，用于告知服务器客户端环境、资源偏好和安全凭证等信息。以下是常见请求头字段及其作用详解：

     ## ⚙️ 一、核心请求头字段

     ### 1. Host
     - **作用**：指定目标服务器的域名和端口号（HTTP/1.1 中必需字段），用于虚拟主机路由。
     - **示例**：`Host: www.example.com:8080`

     ### 2. User-Agent
     - **作用**：标识客户端信息（浏览器类型、操作系统、设备等），服务器据此适配响应（如返回移动端或桌面端页面）。
     - **示例**：`User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) Chrome/120.0.0.0`

     ### 3. Accept 系列字段

     #### Accept
     - **作用**：声明客户端支持的响应内容类型（MIME 类型），如 `text/html` 或 `application/json`，优先级通过 q 参数（0~1）指定。
     - **示例**：`Accept: text/html;q=0.9, application/json;q=0.8`

     #### Accept-Encoding
     - **作用**：指定支持的压缩算法（如 `gzip`），减少传输数据量。
     - **示例**：`Accept-Encoding: gzip, deflate, br`

     #### Accept-Language
     - **作用**：声明首选语言，服务器返回对应语言版本。
     - **示例**：`Accept-Language: en-US, zh-CN;q=0.9`

     ### 4. Connection
     - **作用**：控制连接行为（HTTP/1.1 默认 `keep-alive` 复用连接；`close` 表示请求后关闭）。
     - **示例**：`Connection: keep-alive`

     ### 5. Cache-Control
     - **作用**：指导缓存机制（如 `no-cache` 强制重新验证，`max-age=3600` 缓存1小时）。
     - **示例**：`Cache-Control: max-age=3600`

     ## 🔐 二、安全与状态管理字段

     ### 1. Authorization
     - **作用**：携带身份凭证（如 Bearer Token 或 Basic Auth），用于访问受保护资源。
     - **示例**：`Authorization: Bearer eyJhbGci...`

     ### 2. Cookie
     - **作用**：发送服务器设置的会话信息（如 `sessionId`），维持用户登录状态。
     - **示例**：`Cookie: sessionId=abc123; theme=dark`

     ### 3. Referer
     - **作用**：标记请求来源页面的 URL，用于防盗链、统计流量来源或 CSRF 防护。
     - **示例**：`Referer: https://www.example.com/source-page`

     ## 📦 三、内容与传输控制字段

     ### 1. Content-Type
     - **作用**：指定请求体的数据格式（POST/PUT 请求必需），如 `application/json` 或 `multipart/form-data`（文件上传）。
     - **示例**：`Content-Type: application/json; charset=utf-8`

     ### 2. Content-Length
     - **作用**：声明请求体的字节长度，确保服务器正确解析。
     - **示例**：`Content-Length: 348`

     ### 3. Range
     - **作用**：请求部分内容（用于分块下载或断点续传）。
     - **示例**：`Range: bytes=0-1023`

     ## 🌐 四、其他实用字段

     - **Origin**：标识跨域请求的源站，用于 CORS 安全机制。
       - **示例**：`Origin: https://www.example.com`
     - **X-Requested-With**：标记 AJAX 请求（如 XMLHttpRequest）。
     - **Upgrade-Insecure-Requests**：通知服务器支持 HTTPS 升级（值为 1）。

     ## 💻 五、实际请求头示例

     ```shell
     GET /api/data HTTP/1.1
     Host: api.example.com
     User-Agent: Mozilla/5.0 (Macintosh; Intel Mac OS X 10_15_7)
     Accept: application/json
     Accept-Language: en-US
     Authorization: Bearer eyJhbGci...
     Cache-Control: no-cache
     Cookie: sessionId=xyz789
     Referer: https://www.example.com/dashboard
     ```
