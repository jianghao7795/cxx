SSH 命令的常用参数如下，这些参数能帮助你更灵活、安全地建立连接。

| 参数                                       | 说明                                        | 示例                                          |
| ------------------------------------------ | ------------------------------------------- | --------------------------------------------- |
| `-p <端口>`                                | 指定远程服务器的 SSH 端口（默认为 22）。    | `ssh -p 2222 user@host`                       |
| `-l <用户名>`                              | 指定登录用户名。                            | `ssh -l myname host`                          |
| `-i <私钥文件>`                            | 指定用于身份认证的私钥文件。                | `ssh -i ~/.ssh/id_ed25519 user@host`          |
| `-o <选项>`                                | 设置 SSH 配置选项，可覆盖配置文件中的设置。 | `ssh -o "StrictHostKeyChecking=no" user@host` |
| `-L [本地地址:]本地端口:远程主机:远程端口` | 建立本地端口转发（本地监听，转发到远程）。  | `ssh -L 8080:localhost:80 user@host`          |
| `-R [远程地址:]远程端口:本地主机:本地端口` | 建立远程端口转发（远程监听，转发到本地）。  | `ssh -R 2222:localhost:22 user@host`          |
| `-D [本地地址:]本地端口`                   | 建立动态 SOCKS 代理。                       | `ssh -D 1080 user@host`                       |
| `-N`                                       | 不执行远程命令，仅用于端口转发等。          | `ssh -N -L 8080:localhost:80 user@host`       |
| `-f`                                       | 成功认证后，SSH 进入后台运行。              | `ssh -f -N -L ...`                            |
| `-v`/ `-vv`/ `-vvv`                        | 详细输出模式，用于调试连接问题。            | `ssh -v user@host`                            |
| `-C`                                       | 压缩所有传输的数据。                        | `ssh -C user@host`                            |
| `-q`                                       | 安静模式，抑制大多数警告和诊断信息。        | `ssh -q user@host`                            |
